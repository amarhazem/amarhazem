# üîÅ Development Workflow

1. Move the issue to **In Progress** on the project board.
2. Create a **draft Pull Request** and link it to the issue.
3. Check out the PR locally to start development on the feature branch.
4. **Plan your development**:
   - Outline the required steps
   - Identify impacted areas
   - Define test cases
5. Create or update **automated tests** (unit, integration, or e2e as needed).
6. **Implement** the feature or bugfix.
7. Make sure **all tests pass locally**.
8. Maintain a **100 Lighthouse score** on relevant public-facing pages (Performance, Accessibility, SEO, Best Practices).
9. Update or add **Cursor rules**, if applicable.
10. Update **documentation** (wiki, README, technical notes, etc.) if needed.
11. **Push the final commit** to the branch.
12. Mark the PR as **Ready for Review** (remove draft status).
13. Move the issue to **In Review**.
14. **Address reviewer feedback** and apply requested changes.
15. **Notify the reviewer** once updates are done.
16. **Merge the PR** once approved.
17. Move the issue to **Done**.
18. **Delete the local and remote branch**.
19. Take note of any **friction points** to bring up in the next retrospective.
